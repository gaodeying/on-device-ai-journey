**🚀 90 天端侧 AI 面试冲刺计划（紧扣 JD 定制版）**

#### **第一阶段：重塑技术栈与 AI 基础 (第 1 \- 3 周)**

**目标对应 JD：** 精通 Python、C++，熟悉深度学习框架基础（字节/智元基本要求）。

* **你的现状：** 熟悉移动端，可能 Java/TS/Objective-C 用得多。必须立刻激活 C++（特别是 C++11/14 现代语法）和 Python。  
* **任务清单：**  
  * **Week 1：Python 与 PyTorch 扫盲。** 不要学怎么训练大模型，学怎么**加载、查看、保存**模型。理解 Tensor（张量）的维度、数据类型（FP32 vs FP16）。  
  * **Week 2：重拾 C++ 与 NDK。** AI 在端侧落地 90% 靠 C++。复习智能指针、内存对齐，写几个简单的 JNI/NDK 接口跑在 Android 上。  
  * **Week 3：AI 结构认知。** 搞懂 Transformer 架构和 CNN（如 ResNet/MobileNet）的基础结构。不用默写公式，但要知道什么是 Attention、什么是卷积层，因为后面量化要用。  
* **学习资料：**  
  * PyTorch 官方教程：*Deep Learning with PyTorch: A 60 Minute Blitz*  
  * 《C++ Primer》重点看内存管理和模板部分。

#### **第二阶段：端侧框架与模型转换打通 (第 4 \- 7 周)**

**目标对应 JD：** 熟悉 TFLite、ONNX Runtime，主导端侧 AI 框架开发，解决推理执行（字节 JD 第 2/3 条）。

* **核心动作：把云端模型变成端侧能跑的文件。**  
* **任务清单：**  
  * **Week 4：死磕 ONNX (Open Neural Network Exchange)。** 这是目前的行业标准。学习如何使用 PyTorch 的 torch.onnx.export 将模型导出。学会使用 **Netron** 软件查看网络拓扑图（面试必问）。  
  * **Week 5：TFLite 与 MediaPipe。** 把导出的模型转为 .tflite 格式。理解 TFLite 的解释器（Interpreter）工作原理。  
  * **Week 6：ONNX Runtime 移动端部署。** 用 C++ 在你的 Android/iOS 手机上通过 ONNX Runtime 跑通一个基础的视觉模型（比如 YOLO）或极小的大模型。  
  * **Week 7：端侧算力认知（CPU/GPU/NPU）。** 对应字节 JD 第 4 条。研究 Android 的 NNAPI / AICore 和 Apple 的 Core ML。弄懂什么时候用 GPU 加速，什么时候该切 CPU。  
* **学习资料：**  
  * ONNX 官方文档：*ONNX Python API / C++ API*  
  * GitHub 实战：搜索并跑通开源项目 ncnn-android-yolov8 或 onnxruntime-mobile。

#### **第三阶段：高级调优与量化 (第 8 \- 10 周)**

**目标对应 JD：** 模型量化、图优化、压缩工具，配合模型量化/剪枝/蒸馏（字节/智元核心痛点）。

* **核心动作：把模型变小、变快，且不损失太多精度。这是面试拿高薪的关键。**  
* **任务清单：**  
  * **Week 8：量化 (Quantization) 原理深挖。** 彻底搞懂 PTQ（训练后量化）和 QAT（量化感知训练）。理解 FP32 变 INT8 时，Scale 和 Zero-point 是怎么算的。  
  * **Week 9：动手实操量化。** 对应智元 JD 第 5 条。使用 torchao (智元提到了这个) 或 llama.cpp，把一个 Llama-3-8B 压缩成 4-bit 并测试推理速度和内存占用。  
  * **Week 10：性能 Profiling。** 对应字节 JD 第 3 条。学习使用 torch.profiler 分析模型在哪一层耗时最长；使用 Android Studio Profiler 分析 C++ 层的内存和 CPU 占用。

#### **第四阶段：项目包装与面试突击 (第 11 \- 12 周)**

* **打造“杀手级”开源项目（写进简历）：**  
  * *项目建议：* 基于 C++ 和 NDK，利用 ONNX Runtime 或 ExecuTorch，在 Android 端本地跑通一个量化后的 LLM（如 Qwen-1.5B），并接入系统原生语音 API。  
  * *指标数据（必须体现在简历中）：* 写明“通过 INT4 量化将模型体积减少 70%，内存峰值控制在 1.5GB 内，首字延迟 (TTFT) 优化至 200ms 以下”。  
* **面试话术转换：** 强调你的 11 年终端经验能保障 AI 模型在复杂系统下的**稳定性、低功耗和高良率**，这是纯算法工程师极度缺乏的。
